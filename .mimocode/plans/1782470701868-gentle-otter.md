# Plan : Personnalisation Sectorielle des Sites Générés

## Problème
Les sites web générés pour chaque lead sont ~70% génériques. Un restaurant et un plombier ont la même structure HTML, mêmes icônes, mêmes formulaires, mêmes stats. Il faut que chaque site **reflète visuellement et fonctionnellement** le secteur d'activité.

---

## Phase 1 : Registre Centralisé `SECTOR_CONFIG` (Fondation)

**Nouveau fichier** : `src/lib/sectorConfig.ts`

Créer un registre centralisé qui mappe chaque secteur → configuration complète :

```typescript
interface SectorConfig {
  // Identité visuelle
  schemaOrg: string;           // "Restaurant", "AutoRepair", "HealthAndBeautyBusiness", etc.
  heroBadgeIcon: string;       // Lucide icon name
  heroBadgeText: { fr: string; en: string };
  
  // Navigation sectorielle
  navItems: Array<{ label: { fr: string; en: string }; href: string }>;
  
  // Icônes de services (6 par secteur, noms Lucide)
  serviceIcons: string[];      // ["utensils", "wine", "chef-hat", "concierge-bell", "calendar", "users"]
  
  // Stats sectorielles
  stats: Array<{ value: string; label: { fr: string; en: string } }>;
  
  // Formulaire de contact dynamique
  formFields: Array<{ name: string; type: string; placeholder: { fr: string; en: string }; required: boolean }>;
  
  // Horaires par défaut (fallback si lead.hours absent)
  defaultHours: { weekdays: string; saturday: string; sunday: string };
  
  // Badge "à propos"
  aboutBadge: { value: string; label: { fr: string; en: string } };
  
  // Sections visibles/invisibles
  sections: { process: boolean; whyUs: boolean; stats: boolean; gallery: boolean };
  
  // Style d'animation
  animationStyle: 'default' | 'energetic' | 'elegant' | 'minimal' | 'warm';
  
  // Textes UI sectoriels
  ui: {
    svcTitle: { fr: string; en: string };
    svcDesc: { fr: string; en: string };
    contactTitle: { fr: string; en: string };
    ctaText: { fr: string; en: string };
  };
}
```

**Secteurs à couvrir** : restaurant, coiffeur/salon, plomberie, electricien, garage/auto, médecin/dentiste, avocat, spa/bien-être, fitness, nettoyage, jardin/paysage, + default

### 2a. Fix `imageFilters.ts` — Filtrage sectoriel-aware
**Fichier** : `src/lib/imageFilters.ts`

## Phase 4 : Animations Sectorielles

**Fichier** : `src/lib/ultimateTemplate.ts` (CSS dans `buildUltimateHTML`)

5 styles d'animation basés sur le secteur :

| Style | Secteurs | Caractéristiques |
|-------|----------|------------------|
| `energetic` | restaurant, fitness | Transitions rapides, scale-up, bounce léger |
| `elegant` | coiffeur, spa, avocat | Fade-in doux, transitions lentes, opacité |
| `minimal` | médecin, nettoyage | Pas d'animation décorative, reveal simple |
| `warm` | jardin, plomberie | Slide-in latéral, légère rotation |
| `default` | garage, electricien | Révélation standard actuelle |

- Chaque style = une classe CSS (`anim-energetic`, `anim-elegant`, etc.)
- Les decorations (cercles, diamants, lignes) varient aussi : elegant → moins de decorations, energetic → plus de decorations

2. **Batch test** : Lancer la génération en lot sur 5+ leads de secteurs différents — pas d'erreur

3. **Build test** : `npm run build` — pas d'erreur TypeScript
